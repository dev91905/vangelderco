

## Kill the raw PDF generator — use client-side screenshot-to-PDF instead

### The problem

The current PDF is generated by an edge function (`export-diagnostic-pdf/index.ts`) that manually constructs raw PDF operators with Helvetica Type1 fonts. This approach:

1. **Cannot render Unicode** — bullets (`•`), middots (`·`), checkmarks (`✓`), arrows (`→`), and em-dashes (`—`) all display as mojibake (`â€¢`, `Â·`, `âœ"`, etc.)
2. **Has zero visual design** — no rounded corners, no cards, no score ring, no chips, no colored backgrounds. Just flat text and rectangles.
3. **Wastes 60% of page 1** as blank whitespace
4. **Duplicates layout logic** — the beautiful `DiagnosticReport.tsx` component already exists and renders perfectly in the browser

The fix is simple: capture the already-rendered report as an image, convert it to a PDF on the client side. Same gorgeous design, zero duplication.

### Approach: Client-side html2canvas + jsPDF

Instead of the edge function reconstructing the report from scratch with primitive PDF operators, we'll:

1. **Install `html2canvas` and `jspdf`** as client dependencies
2. **Capture the rendered DiagnosticReport component** — it's already visible in the admin detail panel, so we screenshot it at high resolution
3. **Convert the canvas to a multi-page PDF** with proper margins and page breaks
4. **Delete the edge function** `supabase/functions/export-diagnostic-pdf/` — it's no longer needed

### Technical details

**New dependencies**: `html2canvas`, `jspdf`

**`src/pages/AdminSubmissions.tsx`** — replace `handleExportPdf`:
- Instead of calling the edge function, use `html2canvas` to capture the `.diagnostic-report` element at 2x scale
- Convert the canvas to a jsPDF document, handling multi-page overflow
- Download the resulting PDF blob directly

**`supabase/functions/export-diagnostic-pdf/index.ts`** — delete entirely

**`DiagnosticReport.tsx`** — minor tweaks:
- Add an `id` attribute to the report wrapper (e.g., `id="diagnostic-report-capture"`) for easy targeting
- Ensure `pageBreakAfter: "always"` on page 1 section translates to a clean split in the canvas capture (may need to render pages separately for a clean two-page PDF)

### What the user gets

The exported PDF will look exactly like what they see in the admin panel — score ring, colored dimension bars, gap cards with orange accents, measurement columns with green/orange backgrounds, pill chips, and proper CTA buttons. No encoding issues, no flat text dumps.
